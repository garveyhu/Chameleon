# v0.7 升级指南

> 适用范围：v0.6.x → v0.7.0
> 预计停机时长：< 2 min（alembic upgrade + uv sync）
> 风险等级：**低**（无 breaking；3 个 forward-only changeset；老数据全兼容）

## 一、必做步骤

### 1. 跑数据库迁移

```bash
cd backend
uv run alembic upgrade head
```

按顺序执行 3 个新 changeset：

| Revision | 作用 |
|----------|------|
| `p21_w35_eval_templates` | 新建 `eval_templates`；`eval_jobs` 加 `template_id` (FK SET NULL) + `template_version_frozen` |
| `p21_w37_eval_scores` | `dataset_run_items` 加 `eval_scores` JSONB 列 |
| `p21_w37_kb_consistency` | 新建 `kb_consistency_reports`；`chunks` 加 `quarantined` Boolean + `quarantine_reason` String(64) + `ix_chunks_quarantined` 索引 |

回滚：每个 changeset 带 `downgrade()`，逐条 `alembic downgrade -1`。

**注意**：
- `p21_w37_kb_consistency` 在 chunks 大表上加列；超百万行业务数据建议在低峰窗口执行。
- 老 chunks 的 `quarantined` 默认 `false`；老 eval_jobs 的 `template_id` 为 NULL（不绑模板时行为不变）。

### 2. 装依赖

```bash
cd backend
uv sync
```

无新增 Python 顶层依赖（RAGAS 4 算子本地实现，不引 ragas 包）。

### 3. 启动后端

```bash
uv run uvicorn chameleon.app.main:app --host 127.0.0.1 --port 7009
```

启动日志会看到 sandbox runtimes / scheduler / registry 与 v0.6 一致；P21 各功能不需启动期初始化（按需调用）。

### 4. 更新前端

```bash
cd frontend && yarn install && yarn build
```

无新 npm 依赖；前端新增 `/datasets` 与 `/conversations` 模块路由（自动 glob 注册）；KB 详情页加「一致性」tab。

## 二、新功能启用

### 2.1 Dataset 采样 + PII 脱敏（P21.1）

```bash
# 从 call_log 一键采样到 dataset
curl -X POST "http://localhost:7009/v1/admin/datasets/{ds_id}/sample-from-logs" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_key": "qwen-chat",
    "limit": 50,
    "pii_strategy": "mask",
    "success": true,
    "include_response_as_expected": true
  }'

# 手工 bulk-import（≤ 1000 条/次）
curl -X POST "http://localhost:7009/v1/admin/datasets/{ds_id}/items/bulk-import" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "items": [{"input_payload":{"q":"什么是 RAG？"}, "expected_output":{"answer":"..."}}],
    "pii_strategy": "mask"
  }'
```

PII 策略：
- `mask`（默认）：email/phone/id_card 替换为 `<EMAIL>` / `<PHONE>` / `<ID>` 占位符
- `drop`：含任意 PII 的整条 item 跳过
- `keep`：保留原文（仅明确无 PII 时使用）

### 2.2 EvalTemplate + RAGAS（P21.2）

```bash
# 建模板（version=1）
curl -X POST "http://localhost:7009/v1/admin/eval-templates" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "RAG-4 维",
    "metrics": [
      {"name":"faith", "algorithm":"ragas_faithfulness", "weight":0.3},
      {"name":"rel",   "algorithm":"ragas_answer_relevance", "weight":0.3},
      {"name":"prec",  "algorithm":"ragas_context_precision", "weight":0.2},
      {"name":"recall","algorithm":"ragas_context_recall", "weight":0.2}
    ]
  }'

# 跑 dataset 时联动 template
curl -X POST "http://localhost:7009/v1/admin/datasets/{ds_id}/run" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"run with template","eval_template_id": <tmpl_id>}'

# 查评分分布
curl "http://localhost:7009/v1/admin/datasets/runs/{run_id}/score-distribution?threshold=0.5&buckets=10" \
  -H "Authorization: Bearer $ADMIN_TOKEN"
```

**红线提醒**：
- template `update` → version += 1，老 EvalJob 引用 `template_version_frozen` 不变（行为可预期）
- builtin 4 个 RAGAS 算子注册表只读；自定义 metric 走 EvalTemplate.metrics[].config 字段

### 2.3 KB 一致性扫描 + 修复（P21.3）

```bash
# 扫描（标 quarantined，不物理删）
curl -X POST "http://localhost:7009/v1/admin/kbs/{kb_id}/consistency-reports/scan" \
  -H "Authorization: Bearer $ADMIN_TOKEN"

# 查历史
curl "http://localhost:7009/v1/admin/kbs/{kb_id}/consistency-reports" \
  -H "Authorization: Bearer $ADMIN_TOKEN"

# 一键修复（仅 done/fixed 状态可触发；物理删 quarantined chunks）
curl -X POST "http://localhost:7009/v1/admin/kbs/{kb_id}/consistency-reports/{report_id}/repair" \
  -H "Authorization: Bearer $ADMIN_TOKEN"
```

UI 入口：`/kbs/:id` → 「一致性」tab。

**红线**：scan 只标 `quarantined=True`，不物理删；repair 必须 admin 显式触发（端点单独）+ 仅 `done`/`fixed` 状态可执行。

### 2.4 对话树（P21.4）

UI 入口：`/conversations` 列表 → `/conversations/:sessionId` 详情。

操作：
- assistant 消息 hover → `RefreshCw` 按钮：**regenerate** 创建兄弟分支
- user 消息 hover → `Edit3` 按钮：**edit-and-resend** 编辑后新建 user + assistant 分支
- 每条消息上的 `◀ N/M ▶` 切支器：手动切换查看其它兄弟分支

API：
```bash
# regenerate
curl -X POST "http://localhost:7009/v1/conversations/{sid}/messages/{mid}/regenerate" \
  -H "Authorization: Bearer $APP_KEY"

# edit-and-resend
curl -X POST "http://localhost:7009/v1/conversations/{sid}/messages/{mid}/edit-and-resend" \
  -H "Authorization: Bearer $APP_KEY" \
  -H "Content-Type: application/json" \
  -d '{"new_content": "改后的问题"}'
```

**红线**：regenerate / edit 都不删老消息；新消息走 `parent_message_id` 形成兄弟分支；UI 显式 BranchSwitcher 切支（不偷偷切回老分支）。

## 三、回滚

按相反顺序：

```bash
# 前端
git checkout v0.6.0 -- frontend && yarn install && yarn build

# 后端
cd backend
uv run alembic downgrade -3      # 回滚 3 个 P21 changeset
git checkout v0.6.0 -- chameleon-{core,api,system,app,providers}
uv sync
```

回滚后：
- `eval_templates` / `kb_consistency_reports` 表被 drop
- `eval_jobs.template_id` / `dataset_run_items.eval_scores` / `chunks.quarantined`+`quarantine_reason` 列被 drop
- 所有 P21 数据**不可恢复**
