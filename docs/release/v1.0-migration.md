# v1.0 升级指南

> 适用范围：v0.7.x → v1.0.0
> 预计停机时长：< 3 min（alembic upgrade + uv sync + frontend rebuild）
> 风险等级：**低**（无 breaking；4 个 forward-only changeset）

## 一、必做步骤

### 1. 跑数据库迁移

```bash
cd backend
uv run alembic upgrade head
```

按顺序执行 4 个新 changeset：

| Revision | 作用 |
|----------|------|
| `p22_w41_audit_cost` | audit_logs 加 workspace_id (FK) + session_id；call_logs 加 cost_usd NUMERIC(12,6)；新表 model_pricing |
| `p22_w44_workflow_versioning` | graphs 加 published_spec / published_version / published_at |
| `p22_w45_multimodal_kb` | documents.kind / chunks.kind / chunks.source_url + ix_chunks_kind |
| `p22_w47_app_templates` | 新表 app_templates（应用市场模板） |

回滚：每个 changeset 带 `downgrade()`。

### 2. 装依赖

```bash
cd backend
uv sync
```

无新 Python 顶层依赖（已在 v0.6/v0.7 装齐 docker / pynacl 等）。

### 3. 启动后端

```bash
uv run uvicorn chameleon.app.main:app --host 127.0.0.1 --port 7009
```

启动日志会看到：
- `model_pricing seeded | count=11`（首次启动 seed 默认价目）
- 沿用 v0.6 / v0.7 的 scheduler / sandbox / registry 启动日志

### 4. 更新前端

```bash
cd frontend && yarn install && yarn build
```

无新 npm 依赖；新增页面：
- `/dashboard/cost` 成本统计
- `/traces/:requestId` 独立 trace 详情
- `/marketplace/templates` 应用模板市场

### 5. 装 SDK（可选）

Python：

```bash
pip install chameleon-sdk
```

TypeScript：

```bash
npm install @chameleon/sdk
```

详见 `docs/sdk/python.md` 与 `docs/sdk/typescript.md`。

## 二、新功能启用

### 2.1 Cost dashboard

启动期自动 seed 11 个主流模型价目。之后 `record_call(model_code="gpt-4o-mini", ...)` 自动按价目算 cost_usd。

访问 `/dashboard/cost` 看：
- 区间总成本 / 调用次数 / 平均单次（卡片）
- 时序柱图（hour/day 桶）
- 多维 top-N 表（agent_key / app_id / session_id 三选）

改价目（不溯源）：

```bash
curl -X POST http://localhost:7009/v1/admin/model-pricing \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model_code": "gpt-4o",
    "effective_from": "2027-04-01T00:00:00Z",
    "prompt_price_per_1k": 0.0030,
    "completion_price_per_1k": 0.012,
    "currency": "USD"
  }'
```

旧 call_log 的 cost_usd 不变（红线：可重放）。

### 2.2 OTLP 摄入（外部 trace 落 Chameleon）

```python
from chameleon_sdk import Client

client = Client(api_key="<app-key>", base_url="http://localhost:7009")

with client.trace(name="rag-pipeline") as trace:
    with trace.span("retrieve", observation_type="retriever") as sp:
        sp.set_attribute("kb_id", "demo")
    with trace.span("llm-call", observation_type="generation") as sp:
        sp.set_model("gpt-4o-mini")
        sp.set_usage(prompt_tokens=200, completion_tokens=100)

client.flush()  # POST /v1/otel/v1/traces
```

OTLP 端点强制 app_id 鉴权（X-API-Key 或 Authorization: Bearer），无 token 直接 401。

### 2.3 Workflow draft/published

```bash
# 发布当前 draft → freeze 到 published_spec
curl -X POST "http://localhost:7009/v1/admin/graphs/{id}/publish" \
  -H "Authorization: Bearer $ADMIN_TOKEN"

# 之后改 draft（POST /update）不影响 published_spec
# 要发新版 → 再 POST /publish；published_version += 1
```

UI：graph 编辑器顶部状态条 + 发布按钮 + confirm。

### 2.4 hybrid 6 步 + Reranker + VLM

3 个 mode 切换：

```python
# kb.recall_mode 改 'hybrid' 即可
# 后端 service 调 HybridPipeline；admin 可加 reranker hook
```

VLM caption（image 入 KB）：

```python
from chameleon.core.retrieval import generate_caption

caption = await generate_caption(
    "https://example.com/img.png",
    vlm=my_vlm_client,  # 可选；缺时 fallback 到 fallback_text 或 filename
)
# 写入 chunk content；chunks.kind='image'；source_url=URL
```

### 2.5 应用市场 templates

admin 提交（默认 verified=False）：

```bash
curl -X POST http://localhost:7009/v1/admin/app-templates \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "客服 Agent 模板",
    "category": "agent",
    "spec_json": {"agent_key": "customer-svc"}
  }'

# 审核通过
curl -X POST http://localhost:7009/v1/admin/app-templates/{id}/verify \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"verified": true}'
```

普通用户 list 默认仅返 verified=True。

## 三、回滚

```bash
# 前端
git checkout v0.7.0 -- frontend && yarn install && yarn build

# 后端
cd backend
uv run alembic downgrade -4
git checkout v0.7.0 -- chameleon-{core,api,system,app,providers}
uv sync
```

回滚后所有 v1.0 数据（app_templates / model_pricing / cost_usd / published_spec / chunks.kind 等）**不可恢复**。

## 四、API 兼容承诺

v1.0 后：
- **public API** 进入 deprecation policy：废弃前保留 1 minor 版本
- **SDK** 同上（chameleon-sdk Python、@chameleon/sdk TypeScript）
- **内部 service / private API** 仍可 break，不在承诺范围

deprecated 项标 `Deprecated since vX.Y; will be removed in v(X+1).0` 注释，并在 CHANGELOG 显式列出。
