# v0.5 升级指南

> 适用范围：v0.4.x → v0.5.0 升级
> 预计停机时长：< 2 min（仅 alembic upgrade + uv sync）
> 风险等级：**中**（10 张业务表加列；老 API 全部保留，逻辑兼容）

## 一、必做步骤

### 1. 跑数据库迁移

```bash
cd backend
uv run alembic upgrade head
```

按顺序执行 4 个新 changeset：

| Revision | 作用 |
|----------|------|
| `p19_w17_eval_jobs` | 新建 eval_jobs + eval_job_runs |
| `p19_w19_plugins` | 新建 plugin_instances |
| `p19_w21_workspaces` | 新建 workspaces / teams / memberships / workspace_quotas + 10 张业务表加 `workspace_id` (NULLABLE, FK) + 老数据 backfill 到 default ws (id=1) |
| `p19_w23_message_blocks` | messages 加 `content_blocks` JSONB (NULL = 老纯文本) |

回滚：每个 changeset 都带 `downgrade()`，逐条 `alembic downgrade -1`。

**注意**：`p19_w21_workspaces` 包含数据迁移（UPDATE 10 张业务表的 `workspace_id`），跨百万行业务数据时建议在低峰窗口执行。

### 2. 启动后端

```bash
uv sync   # 拉 apscheduler 等新依赖（如果上 v0.4 之前没装）
uv run uvicorn chameleon.app.main:app --reload
```

启动日志会出现：

```
eval_jobs scheduler started
eval_jobs scheduler reloaded N jobs
plugin builtin seeded | count=3       # 首次启动
workspace_quota reset done | affected=N  # 每天 00:05 UTC 一次
```

第一次跑业务 invoke 时，未配 quota 的 workspace 自动按 NULL 上限放过（不限）。

### 3. 更新前端

```bash
cd frontend && yarn install && yarn build
```

无新增三方依赖；types/store 都是项目内增量。

### 4. 重新登录

`X-Workspace-Id` header 由前端 axios interceptor 自动注入。首次登录后默认 admin 看全量；管理员可在侧边栏顶部的 workspace 切换器选具体 workspace。

## 二、可选 / 推荐

### 启用 Eval 自动化

```bash
# 1. 建 dataset（v0.4 已有）+ 采样
# 2. 建 cron eval job
curl -X POST /v1/admin/eval-jobs \
  -H "Authorization: Bearer $ADMIN_JWT" \
  -d '{
    "job_key": "daily-baseline",
    "name": "Daily 基线回归",
    "dataset_id": 123,
    "cron_expr": "0 2 * * *",
    "judge": "exact_match",
    "alert_config": {
      "kind": "slack",
      "target": "https://hooks.slack.com/services/...",
      "regression_threshold": 0.1,
      "silence_minutes": 60
    }
  }'
```

或者直接用前端 `/eval-jobs` 列表页 → 「新建」按钮。

### 启用 Plugin Hot Loader

```bash
# 列内置 builtin（local / dify / fastgpt 已自动 seed）
curl /v1/admin/plugins -H "Authorization: Bearer $ADMIN_JWT"

# 安装外部 plugin（前提：插件 Python 包已 pip 安装到 venv）
curl -X POST /v1/admin/plugins/install \
  -H "Authorization: Bearer $ADMIN_JWT" \
  -d '{
    "manifest": {
      "name": "openrouter",
      "version": "1.0.0",
      "type": "provider",
      "entrypoint": "openrouter_chameleon.provider:OpenRouterProvider",
      "chameleon_version": ">=0.5",
      "permissions": {"network": true}
    },
    "source": "local"
  }'
```

或前端 `/plugins` 列表 → 「安装插件」按钮粘贴 manifest JSON。

### 启用 Multi-tenant

```bash
# 1. 建 workspace
curl -X POST /v1/admin/workspaces -d '{"workspace_key":"acme","name":"Acme","plan":"pro"}'
# 2. 添加成员
curl -X POST /v1/admin/workspaces/<id>/members -d '{"user_id":<uid>,"role":"member"}'
# 3. 设配额
curl -X POST /v1/admin/workspaces/<id>/quota/update -d '{"token_quota_monthly":1000000,"request_quota_daily":5000}'
```

前端：sidebar 顶部 dropdown → 「新建 workspace」+「管理成员」。

### 启用 Multimodal

无需配置；前端 Playground 输入区已带 Paperclip 上传按钮。后端 `MINIO_ACCESS_KEY` / `MINIO_SECRET_KEY` / bucket 早已配过即可（v0.3 起 KB 上传依赖）。

## 四、回滚

```bash
git checkout v0.4.0
cd backend
for _ in 1 2 3 4; do
  uv run alembic downgrade -1
done
uv run uvicorn chameleon.app.main:app
```

回滚后：
- 已写入的 eval_jobs / plugin_instances / workspaces / memberships / workspace_quotas 数据全部 drop
- 10 张业务表的 `workspace_id` 列删除（数据中的 ws 归属信息丢失）
- messages.content_blocks 列删除（老 plain text content 仍在）

如果想保留数据，回滚前 `pg_dump` 备份 5 张相关表 + 10 张业务表的 `workspace_id` 列。

## 五、新红线（plan §2 P19 立）

- ⛔ Plugin 加载必须 async + 5s 超时；manifest 不允许包含可执行代码；plugin entrypoint 不能指向 `chameleon.core.models.*` / `infra` / `system` / `api` / `app` 等内部模块
- ⛔ Multi-tenant 改 schema 必须 backward-compat（`workspace_id` NULLABLE + default ws backfill）
- ⛔ Eval alert 必须 Redis rate-limit + dedup；告警失败不污染主 trigger 路径
- ⛔ 配额检查走单点中间件（`assert_within_request_quota`），业务路由不分散写
- ⛔ Multimodal image / audio 走 URL，不内嵌 base64
- ⛔ 切换 workspace 强制 `queryClient.clear()` 防跨租户缓存污染

## 六、升级后第一周观察

- **Eval Jobs**：`/eval-jobs/:id` 详情页趋势线是否上涨/下跌；alert 是否真的触达 Slack
- **Plugins**：`plugin_instances` 表非 builtin 行数 → admin 自助装插件意愿
- **Workspaces**：`memberships` 行数；非 admin 用户 `X-Workspace-Id` header 命中率
- **Quotas**：`workspace_quotas.token_used_current_month` 接近 limit 的 workspace 数 → 是否需要联系扩容
- **Files**：MinIO bucket 大小增长曲线；20MB 上限是否需要调；mime 白名单是否要加新类型
- **Multimodal**：`messages.content_blocks IS NOT NULL` 占比 → 多模态消息渗透率
