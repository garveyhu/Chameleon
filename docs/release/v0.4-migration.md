# v0.4 升级指南

> 适用范围：v0.3.x → v0.4.0 升级
> 预计停机时长：< 1 min（仅 alembic upgrade）
> 风险等级：低（无 breaking changes，老 API 全部保留）

## 一、必做步骤

### 1. 跑数据库迁移

```bash
cd backend
uv run alembic upgrade head
```

按顺序执行 5 个新 changeset：

| Revision | 作用 |
|----------|------|
| `p18_w9_graphs` | 新建 graphs / graph_runs / graph_node_runs 三张表 |
| `p18_w12_tools` | 新建 tool_instances 表 |
| `p18_w13_datasets` | 新建 datasets + dataset_items 表 |
| `p18_w13_dataset_runs` | 新建 dataset_runs + dataset_run_items 表 |
| `p18_w15_message_branch` | messages 加 parent_message_id 列（向后兼容 NULL） |

回滚：每个 changeset 都带 `downgrade()`，逐条 `alembic downgrade -1`。

### 2. 启动后端

```bash
uv sync
uv run uvicorn chameleon.app.main:app --reload
```

启动日志会多两个 registry 摘要：

```
Loaded N graph node types: noop, start, end, llm, kb, tool, if_else
Loaded N tools: http, sql
```

第一次跑 graph 时 system app 自动创建（call_logs.app_id FK 需要）。

### 3. 更新前端

```bash
cd frontend && yarn install && yarn build
```

`@xyflow/react@12.10` 是新依赖（React Flow），首次会下载 ~1MB。

## 二、可选 / 推荐

### 启用 GraphEngine

无需配置。默认即可用：
1. 侧边栏 → "工作流" → 新建
2. 拖 LLM / KB / Tool node 连线
3. 点 **Test Run**（不写库，debug）或 **Run**（持久化 + 写 trace）
4. 跑完去 `/call-logs?app_id=system` 点行展开 Tree tab 看嵌套

### 配 Tool

进入 `/v1/admin/tools` 端点（admin 界面 P19 落地 UI）：

```bash
curl -X POST http://localhost:7009/v1/admin/tools \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "tool_key": "http",
    "name": "Public HTTP",
    "config": {"allowed_url_prefixes": ["https://api.example.com/"]}
  }'
```

SQLTool 默认 `enabled=false`，要用须 admin 显式开 + 配 `allowed_db_urls` 白名单。

### 让 LLMNode 调 Tool

GraphEditor 中 LLM 节点的 inspector 里 spec.data 添加：

```json
{
  "tool_keys": ["http"]
}
```

调用时 LLM 决定调 http → 返 tool_calls；后续 ToolNode 收到并执行。

### 跑 Eval

```bash
# 1. 建 dataset
curl -X POST /v1/admin/datasets -d '{"name":"我的测试集"}'
# 2. 从 call_log 采样
curl -X POST /v1/admin/datasets/{id}/sample-from-logs -d '{"app_id":"...","limit":50}'
# 3. 人工标注 expected_output（可选）
curl -X POST /v1/admin/datasets/items/{item_id}/update -d '{"expected_output":{"answer":"金标准"}}'
# 4. 跑一次 run
curl -X POST /v1/admin/datasets/{id}/run -d '{"name":"baseline","judge":"contains"}'
# 5. 对比两个 run
curl -X POST /v1/admin/datasets/runs/compare -d '{"run_ids":[id_a, id_b]}'
```

scores 表会自动写入 `source='eval'` 行，trace tree drawer 也能看到。

### Chunking 实时预览

进入 `/kbs/:id` 详情页 → Header 右上 "✂︎ 切块预览"：
- 左：粘贴原文
- 右：选 mode + 调参（fixed / paragraph / sentence / regex / token）
- 中：实时 chunks 卡片预览（不写库）

确认满意后去「配置」tab 保存到 KB。

## 四、回滚

```bash
git checkout v0.3.x
cd backend
for _ in 1 2 3 4 5; do
  uv run alembic downgrade -1
done
uv run uvicorn chameleon.app.main:app
```

回滚后：
- 已写入的 graph_runs / dataset_runs / tool_instances 数据会随 changeset 一并 drop
- messages.parent_message_id 列删除 —— 不影响老消息（NULL 即可）

如果想保留数据，回滚前先 `pg_dump` 备份 5 张相关表。

## 五、新红线（plan §2 PR #17/22/24 已立）

- ⛔ GraphEngine Node 不共享可变状态 —— 节点只能通过 edges 显式传 data
- ⛔ Tool 不持 db session —— 走依赖注入；防绕过权限
- ⛔ Code Sandbox 永不在主进程跑 —— 占位接口未上线
- ⛔ Dataset 不存原始 PII —— 采样强制脱敏
- ⛔ 不允许 graph 嵌套 graph —— 防爆栈

## 六、升级后第一周观察

- `/graphs/:id/runs` 列表是否有积累 → GraphEngine 实际使用度
- `tool_instances.enabled=true` 数量 → admin 主动配 tool 的意愿
- `dataset_runs.summary.mean_score` 分布 → judge 选择是否合适
- `call_logs` 里 `observation_type='trace'` 与 graph_runs 数量比 → trace 串联是否正常
