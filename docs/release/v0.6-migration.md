# v0.6 升级指南

> 适用范围：v0.5.x → v0.6.0 升级
> 预计停机时长：< 2 min（仅 alembic upgrade + uv sync；docker 镜像 pre-pull 视带宽）
> 风险等级：**低**（无 breaking；2 张新表 + 1 张表加 4 列；老数据全兼容）

## 一、必做步骤

### 1. 跑数据库迁移

```bash
cd backend
uv run alembic upgrade head
```

按顺序执行 2 个新 changeset：

| Revision | 作用 |
|----------|------|
| `p20_w27_plugin_registries` | 新建 `plugin_registries`（远端 registry CRUD + pubkey pinning + entries 缓存） |
| `p20_w29_kb_collections` | 新建 `kb_collections`；`chunks` 加 `collection_id` (FK ondelete=SET NULL) / `index_name` / `qa_question` / `api_endpoint` + 复合索引 |

回滚：每个 changeset 都带 `downgrade()`，逐条 `alembic downgrade -1`。

**注意**：
- `p20_w29_kb_collections` 在 chunks 大表上加列（`ADD COLUMN ... DEFAULT 'chunk'`）；超百万行业务数据建议在低峰窗口执行。
- 老 chunks 的 `collection_id` 默认 `NULL`；ingest pipeline 兼容 NULL 路径，不强制 backfill。
- 新建 KB 时 service 自动 seed 一个 `name='default', collection_type='generic'` 的 collection。

### 2. 装新依赖

```bash
cd backend
uv sync
```

新增 Python 依赖：

| 包 | 版本 | 用途 |
|----|------|------|
| `docker` | `>=7.1` | 沙箱 docker runtime（docker-py SDK） |
| `pynacl` | `>=1.5` | 插件 manifest Ed25519 验签 |

如果生产环境是 alpine / 极简镜像，确保系统装了 `libsodium`（pynacl 依赖；多数 base image 已带）。

### 3. 启动后端

```bash
uv run uvicorn chameleon.app.main:app --reload
```

启动日志会出现：

```
sandbox runtime registered | name=mock | mock=True
sandbox runtime registered | name=docker | mock=False    # 如果 docker daemon 可达
sandbox bootstrap done | runtimes=['mock', 'docker']
```

如 docker daemon 不可达（CI 环境 / 受限服务器）：

```
sandbox runtime register skipped | name=docker | reason=...
sandbox bootstrap done | runtimes=['mock']
```

**注意**：
- 启动期 docker 不可达**不会**阻塞服务，仅 `mock` 可用
- 生产环境（`CHAMELEON_ENV=production`）下 `mock` runtime 会拒绝加载，必须装 docker；
  dev 环境随意

### 4. 更新前端

```bash
cd frontend && yarn install && yarn build
```

无新增 npm 依赖；前端只是新增 `/marketplace` 页 + KB Collections tab + graph 编辑器 agent_debate 节点 UI。

## 二、Sandbox docker 镜像准备（P20.1）

生产用沙箱镜像建议自建：

```dockerfile
# chameleon-sandbox:py3.12
FROM python:3.12-slim
USER nobody:nobody
WORKDIR /workspace
ENTRYPOINT ["sh", "-c"]
```

构建 + 推 registry：

```bash
docker build -t chameleon-sandbox:py3.12 -f sandbox/Dockerfile sandbox/
docker tag chameleon-sandbox:py3.12 registry.example.com/chameleon-sandbox:py3.12
docker push registry.example.com/chameleon-sandbox:py3.12
```

admin 在 `tools_instances` 表配 code-runner 时：

```jsonc
{
  "tool_key": "code-runner",
  "config": {
    "runtime": "docker",
    "image": "registry.example.com/chameleon-sandbox:py3.12",
    "language": "python",
    "timeout_sec": 30,
    "memory_mb": 256,
    "cpu_quota": 0.5,
    "network": "none",
    "max_stdout_bytes": 1048576
  }
}
```

**红线提醒**：
- `network: 'full'` 永远不被接受（`SandboxConfig.__post_init__` 拒绝）
- timeout 上限 600s；memory 上限 4096MB；cpu 上限 4 核
- 用户代码非零退出**不算** tool fail（数据带 `exit_code` + `user_code_failed=True` meta）

## 三、Plugin Marketplace 配置（P20.2）

### 3.1 注册 registry

通过 admin UI `/marketplace` 添加：

| 字段 | 示例 |
|------|------|
| name | "Chameleon Official" |
| registry_url | `https://registry.chameleon.dev` |

或直接 POST：

```bash
curl -X POST http://localhost:8000/v1/admin/marketplace/registries \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Chameleon Official",
    "registry_url": "https://registry.chameleon.dev"
  }'
```

### 3.2 同步 + 安装

```bash
# 拉 index.json，缓存 publisher 公钥与 entries
curl -X POST http://localhost:8000/v1/admin/marketplace/registries/{id}/sync \
  -H "Authorization: Bearer $ADMIN_TOKEN"

# 搜索 + 安装（manifest 必带 Ed25519 签名，pubkey 必在 registry pin 列表）
curl -X GET 'http://localhost:8000/v1/admin/marketplace/search?q=openrouter' \
  -H "Authorization: Bearer $ADMIN_TOKEN"

curl -X POST http://localhost:8000/v1/admin/marketplace/install \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"registry_id": 1, "plugin_name": "openrouter-provider"}'
```

**红线**：
- manifest 签名校验失败 → 直接拒绝安装；不放过"未知签名 + 警告"路径
- publisher 公钥不在 registry pin 列表 → 跳过该 entry（防钓鱼包）

## 四、KB Collection types 启用（P20.3）

### 4.1 老 KB 不受影响

```
collection_id NULL → 走兼容路径，ingest / retrieve 与 v0.5 行为完全一致
```

### 4.2 新建带类型的 KB

通过 admin UI `/kbs/:id` → Collections tab → 新建：

| 类型 | 适合内容 | chunker |
|------|----------|---------|
| `generic` | 普通文档 | 按段落 / 字符长度切 |
| `faq` | QA 对（`## Q: ... \n ## A: ...`） | 每对一 chunk + 填 `qa_question` |
| `wiki` | 长文 markdown 含 `#`/`##` heading | 按 heading 切 + 维护 path |
| `api` | OpenAPI YAML/JSON | 每 endpoint 一 chunk + 填 `api_endpoint` |

**红线**：`collection_type` 一经写入**不可改**；改类型 = 新建 collection 重新 ingest。

### 4.3 通过 API 创建

```bash
curl -X POST http://localhost:8000/v1/admin/kbs/{kb_id}/collections \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "faq-zh",
    "collection_type": "faq",
    "indexes": [{"name": "chunk", "dim": 1536, "enabled": true}]
  }'
```

## 五、Agent 协同启用（P20.4）

graph 编辑器拖 **Agent Debate** 节点到画布：

1. inspector 加 ≥ 2 个 agent（按顺序映射 PROPOSER / CRITIC / JUDGE / critic+N）
2. 配置 `max_rounds (1-10) / timeout (s) / early_stop_on (consensus|max_rounds) / total_budget_tokens`
3. 保存 + Test Run

**红线**：
- `max_rounds` 硬上限 10（防 agents 无限互打）
- `timeout_total_sec` 软超时返当前最佳结果（不抛）
- `total_budget` 跨 agent 共享；耗尽即 stop
- A2A 调用必须传 `trace_id`；最大嵌套深度 `MAX_DEPTH=3` 防递归爆栈

## 六、回滚

按相反顺序：

```bash
# 前端：降级到 v0.5.0 tag
git checkout v0.5.0 -- frontend && yarn install && yarn build

# 后端：回滚 alembic + uv lock
cd backend
uv run alembic downgrade -2     # 回滚 2 个 P20 changeset
git checkout v0.5.0 -- chameleon-{core,api,system,app,providers}
uv sync
```

回滚后：
- `kb_collections` / `plugin_registries` 表被 drop
- `chunks` 表 4 个新列被 drop
- 数据丢失：所有用户在 v0.6 创建的 collections / registries 数据**不可恢复**
