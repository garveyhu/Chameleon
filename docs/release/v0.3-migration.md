# v0.3 升级指南

> 适用范围：v0.2.x → v0.3.0 升级
> 预计停机时长：< 1 min（仅 alembic upgrade）
> 风险等级：低（无 breaking changes，老路由路径仍可工作）

## 一、必做步骤

### 1. 跑数据库迁移

```bash
cd backend
uv run alembic upgrade head
```

会顺序执行 6 个新 changeset：

| Revision | 作用 |
|----------|------|
| `p17_w3_channels` | 新建 channels 表 + 自动回填每个 provider 的 default channel |
| `p17_w4_abilities` | 新建 abilities 矩阵表 |
| `p17_w4_agent_model_code` | agents 表加 `model_code` 字段（NULL 兼容） |
| `p17_w6_observation` | call_logs 加 parent_id / observation_type / completion_start_ms |
| `p17_w6_channels_cascade` | channels.provider_id FK 改 CASCADE（修复 seed 导入路径） |
| `p17_w7_scores` | 新建 scores 表 |

回滚：所有 changeset 都带 `downgrade()`，逐条 `alembic downgrade -1` 即可。

### 2. 启动新版后端

```bash
cd backend
uv sync
uv run uvicorn chameleon.app.main:app --reload
```

启动后会在日志看到 registry summary 多了：

```
Loaded N providers: ...
Loaded N agents: ...
Channels backfilled: M default channels for providers
```

### 3. 更新前端

```bash
cd frontend && yarn install && yarn build
cd embed && yarn install && yarn build
```

embed widget 的 v0.3.0 产物已支持新的 `/feedback` 接口；老 widget bundle 与新后端兼容（不会调 feedback 而已）。

## 二、可选 / 推荐

### 启用路由矩阵（abilities）

v0.3 默认 `routing.use_abilities=false`，调用方按老路径走（agent 直绑 provider）。

要启用路由矩阵：

```bash
curl -X POST http://localhost:7009/v1/admin/settings/value \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"key":"routing.use_abilities","value":"true","group":"general"}'
```

之后调用方传 `model_code` 即可被 router 解析到 channel。原 agent → provider 路径仍可走（feature flag off 时优先）。

### 给关键 channel 配置 priority / weight

进入 Channels 页（admin → AI 配置 → Channels），把生产 channel 的 `priority` 设为 100、`weight` 设为 10；备用 channel `priority=50`、`weight=5`。
失败时 failover wrapper 自动跨 priority 切换。

### 让 widget 反馈写入 scores

不需要任何代码改动 —— 新 widget bundle 加载后自动调 `/v1/embed/{key}/feedback`。
admin 在 call_logs 详情 Tree tab 能看到 source='feedback' 的 ScoreBadge。

### 给 KB 启用 TokenChunker

KB 详情 → 配置 tab → 分块策略选 "按 Token"，单位切到 token。
**重新分块所有文档**：点 "应用并重分块所有文档" 按钮。
注意：重分块期间检索质量下降，新 chunks 完成前不会回显。

### 让用户挑主题色

Settings → 外观 tab，挑 primary / neutral / animation。
偏好持久在浏览器 localStorage（key=`chameleon:preferences`），不上云同步。

## 三、回滚

如果遇到问题：

```bash
# 1. 切回老版本代码
git checkout v0.2.x

# 2. 回滚 6 个新 changeset
cd backend
for _ in 1 2 3 4 5 6; do
  uv run alembic downgrade -1
done

# 3. 重启
uv run uvicorn chameleon.app.main:app
```

风险点：

- 老代码不认 `parent_id` / `observation_type` 列也无所谓（SQLAlchemy 只 select 已知列）—— 不回滚 changeset 也能跑老版本，但 trace tree / scores 数据会"悬空"
- 已写入的 scores 行回滚后会被一并删除（cascade not modeled，scores 表整体 drop）—— 上线后立即出问题且需要回滚 scores 时，先 `pg_dump scores -t scores` 备份

## 四、新 API 一览

| Method | Path | 说明 |
|--------|------|------|
| GET | `/v1/admin/channels` | channels 列表 + filter |
| POST | `/v1/admin/channels` | 创建 channel（api_key 加密落库） |
| POST | `/v1/admin/channels/{id}/update` | 更新 |
| POST | `/v1/admin/channels/{id}/delete` | 软删 |
| GET | `/v1/admin/channels/{id}/health` | 实时健康（fail_count / EWMA / used_quota） |
| GET | `/v1/admin/abilities` | abilities 列表 + filter |
| POST | `/v1/admin/abilities` | 创建 / 更新 / 删除（同 channel 风格） |
| GET | `/v1/admin/call-logs/{request_id}/tree` | 嵌套 observation 树 |
| GET | `/v1/admin/scores?call_log_id=…` | 列 scores |
| POST | `/v1/admin/scores` | admin 写 score（人工标注） |
| POST | `/v1/embed/{key}/feedback` | widget 反馈（自动 source='feedback'） |

## 五、关键观察点（升级后第一周）

- `routing.use_abilities=true` 切换后 24h 内观察：
  - `call_logs` 里 `success rate` 是否回落（路由命中率问题）
  - failover 日志 `provider failed | switch to next` 频率
- TokenChunker 文档：先在低优 KB 上试切，观察 retrieval 召回率是否提升 / 退化
- scores 表写入 QPS 监控（widget 反馈量级 × scores 写 / s）—— v0.3 是同步写 + 短事务，QPS 高时考虑 v0.4 改异步 batch
