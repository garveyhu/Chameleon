## Agent A · Workflow 引擎

### SSE 事件 kind（`SSEEventKind`，前端 TS 需镜像）

GraphEngine 流式执行新增的 graph 域事件 kind（dotted 命名做命名空间），wire 上是
扁平 dict `{kind: payload}`，由 `sse_response` 序列化为 `data: {json}\n\n`，流末 `data: [DONE]`：

| kind | 触发 | payload |
|------|------|---------|
| `graph.started` | 整图开始（1 次） | `{ graph_id: number, run_id: string }` |
| `graph.node.started` | 节点开始 | `{ node_id, node_type, name, status: "running" }` |
| `graph.node.delta` | 节点流式 token（A2） | `{ node_id, delta: string }` |
| `graph.node.finished` | 节点成功 | `{ node_id, node_type, name, status: "success", duration_ms, output }` |
| `graph.node.failed` | 节点失败 | `{ node_id, node_type, name, status: "failed", duration_ms, error: { type, message } }` |
| `graph.finished` | 整图结束（1 次） | `{ status: "success"\|"failed", duration_ms, node_count, output?, error? }` |

约束：`graph.node.failed` 出现后该节点不再有 finished；整图最终一定有一个 `graph.finished`。
`graph.node.delta` 只由支持流式的节点（当前 LLMNode）在 `started` 与 `finished` 之间按
node_id 多次发出，前端按 node_id 累积渲染。

### LLMNode spec.data 扩展（A2）

LLM 节点新增可选配置（`graphs` 的 spec.nodes[].data）：

| 字段 | 说明 |
|------|------|
| `memory_window` | int ≥ 1；input 含 `history: [{role,content}]` 时保留最近 N 条多轮上下文 |
| `tool_keys` | `string[]`；启用 agentic 多轮 tool_call（模型选 tool → 执行 → 回填 → 再调） |
| `max_tool_rounds` | int [1,8]，默认 3；agentic 循环最多轮数（防无限调用） |

LLM 节点 output 新增：`tool_rounds`（每轮 `{round, assistant, tool_calls, tool_results}`）、
`rounds_used`。正式 `run`（持久化）时，tool_rounds 串成 LLM 节点的子观测
（`parent_id` = 节点 request_id，`observation_type="tool"`），在 trace tree 里嵌套展示。

### 新节点类型 `iteration`（A4）

对列表逐元素跑子图（map / loop）。spec.nodes[].data：

| 字段 | 说明 |
|------|------|
| `body` | 必填，子图 GraphSpec（含自己的 start/end），对每个 item 跑一遍 |
| `items_path` | 可选，从 input dict 取数组的 dot 路径；不填则 input 本身是 list |
| `item_input_key` | 可选，子图 input = `{item_input_key: item}`；不填则 item 直接作子图 input |
| `early_stop` | 可选，if_else 表达式；对每个 item 的子图输出 truthy 则停（强制串行）|
| `max_iterations` | 可选，默认 100，cap 1000 |
| `concurrency` | 可选，>1 并行（无 early_stop 时），保序收集 |

output：`{ items: [...], count, stopped_early, stopped_at }`。

### 新节点类型 `parallel`（A5）

fork-join：同一 input fork 给 N 条不同 branch 子图并发跑，再 join。spec.nodes[].data：

| 字段 | 说明 |
|------|------|
| `branches` | 必填，`[{key, body: GraphSpec}, ...]`（2–20 条，key 唯一）|
| `join_strategy` | `collect`（默认，等全部成功）/ `merge`（浅合并各分支 dict）/ `race`（最先成功者胜，取消其余）|
| `concurrency` | 可选，默认 = 分支数（全并发）|

output（collect/merge）：`{ branches: {key: out}, merged?: {...}, branch_runs: [...] }`；
race：`{ winner, output, branch_runs }`。`branch_runs` 每条带 `started_offset_ms` +
`duration_ms`（相对节点起点），供 Trace Gantt 渲染并发重叠。正式 `run` 时 branch_runs
串成节点子观测（`observation_type="span"`）。

### 新节点类型 `human_input` + 暂停/恢复（A6）

`human_input` 节点让工作流跑到此暂停、等人工回填后从断点恢复。spec.nodes[].data：

| 字段 | 说明 |
|------|------|
| `prompt` | 可选，给审核人的提示 |
| `schema` | 可选，期望输入的 JSON schema（前端渲染表单）|
| `timeout_seconds` | 可选，[1, 30 天]；超时未回填则该 run 自动 failed |

**[SCHEMA-CHANGE]** 新增表 `human_input_pending`（migration `p23_w52_human_input_pending`，
合并 train 中 re-parent 到 C 链尾 `p23_w51_channel_keys`）。

正式 `POST /{graph_id}/run` 命中 human_input：graph_run.status=`paused`，落一行 pending。

- `GET /v1/admin/graphs/pending?status=pending` → 列待回填断点
  `[{ id, graph_run_id, node_id, prompt, input_schema, node_input, timeout_at, ... }]`
- `POST /v1/admin/graphs/runs/{run_id}/resume` body `{ value: {...} }` → 回填恢复，
  返回 GraphRunDetail（可能再次 paused 或终态）。权限 `graphs:execute`。

超时清扫：APScheduler 每 5 分钟扫 `timeout_at < now` 且仍 pending 的断点 → 标 timeout +
对应 run failed。

### `POST /v1/admin/graphs/{graph_id}/test-run/stream`（A1）

SSE 流式 Test Run（**不落** graph_runs，debug 用）。权限 `graphs:execute`。

请求体（同 `test-run`）：

```json
{ "input": { "query": "..." } }
```

响应：`text/event-stream`，按上表逐条推 graph 域事件，流末 `[DONE]`。示例：

```
data: {"graph.started":{"graph_id":1,"run_id":"testrun-1-..."}}
data: {"graph.node.started":{"node_id":"s","node_type":"start","status":"running"}}
data: {"graph.node.finished":{"node_id":"s","node_type":"start","status":"success","duration_ms":1,"output":{...}}}
...
data: {"graph.finished":{"status":"success","duration_ms":42,"node_count":3,"output":{...}}}
data: [DONE]
```

非流式 `POST /v1/admin/graphs/{graph_id}/test-run`（已存在）保持不变。

## Agent B · RAG 检索深化

### `POST /v1/admin/kbs/{kb_id}/search` —— hit-test（score breakdown）

KB 检索调试 playground。v1.1 起统一走 `core.retrieval.pipeline`，返回每条命中的
**分项得分**（供 D5 三栏 hit-test 分项条形图渲染）。

**Request body**（`SearchRequest`）：

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `query` | string | 必填 | 检索词 |
| `top_k` | int (1-50) | 5 | 返回条数 |
| `min_score` | float (0-1) | 0.0 | 最小分阈值 |
| `doc_ids` | int[] \| null | null | 限定文档 |
| `tags` | string[] \| null | null | 按文档 tag 过滤 |
| `mode` | `vector`\|`hybrid`\|`keyword` \| null | null（走 KB.recall_mode） | 召回模式 |
| `include_images` | bool \| null | null（走 KB.meta.include_images） | 放行图片 chunk（多模态） |
| `multi_query` | int (0-5) | 0 | >1 时 LLM 改写 N 变体多路融合（B1） |
| `hyde` | bool | false | HyDE 假设答案召回（B2） |

**Response**（`Result<SearchHitItem[]>`）—— `SearchHitItem`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `chunk_id` | int | |
| `doc_id` | int | |
| `seq` | int | |
| `content` | string | chunk 文本（图片为 caption） |
| `score` | float | 最终排序分（融合/rerank 后） |
| `document_title` | string | |
| `kind` | string | `text` \| `image`（B5 多模态） |
| `source_url` | string \| null | 图片 chunk 的资源 URL；文本为 null |
| `vector_score` | float \| null | 向量召回原始余弦相似度 |
| `bm25_score` | float \| null | BM25(ts_rank) 归一化分 |
| `rerank_score` | float \| null | reranker 重排分（启用 reranker 才有） |

> 分项缺省为 `null`：仅向量命中的 chunk 无 `bm25_score`；未启用 reranker 时
> `rerank_score` 全 null。D5 渲染时按 null 隐藏对应条形。

**示例响应**：

```json
{
  "code": 0,
  "success": true,
  "message": "ok",
  "data": [
    {
      "chunk_id": 1012, "doc_id": 22, "seq": 3,
      "content": "……", "score": 0.92, "document_title": "架构说明",
      "kind": "text", "source_url": null,
      "vector_score": 0.91, "bm25_score": 0.42, "rerank_score": 0.92
    },
    {
      "chunk_id": 1099, "doc_id": 25, "seq": 1,
      "content": "[image] topology.png", "score": 0.78, "document_title": "拓扑图",
      "kind": "image", "source_url": "kb_uploads/3/1099.bin",
      "vector_score": 0.78, "bm25_score": null, "rerank_score": null
    }
  ]
}
```

### Reranker 配置（KB 级，默认关）

reranker 走 `KB.meta.reranker` 配置（无则关）。形态：

```json
{ "type": "bge|cohere|local_dedupe|llm_judge|none",
  "model": "bge-reranker-v2-m3", "base_url": "http://...", "api_key": "...",
  "top_n": 5 }
```

`include_images` 默认走 `KB.meta.include_images`。两者均可被 search 请求的同名字段覆盖。
